Componente

Button

Botón semántico aplicado con el selector button[neu-button]. Usa Angular Signals (input(), computed()) para gestionar el estado de forma reactiva.

ViewEncapsulation.None ChangeDetection.OnPush Signals

Variantes

El input variant controla el estilo visual.

<button neu-button variant="primary">Primary</button>
<button neu-button variant="secondary">Secondary</button>
<button neu-button variant="outline">Outline</button>
<button neu-button variant="ghost">Ghost</button>
<button neu-button variant="danger">Danger</button>

Tamaños

El input size acepta sm, md (default) y lg.

Estados con Signals

El estado loading es un signal() reactivo. Pulsa para simular una petición async. 

// TypeScript — Signal reactivo
readonly isLoading = signal(false);

simulateLoad(): void {
  this.isLoading.set(true);
  setTimeout(() => this.isLoading.set(false), 2500);
}

// Template
<button neu-button [loading]="isLoading()" (click)="simulateLoad()">
  {{ isLoading() ? 'Procesando...' : 'Enviar' }}
</button>
