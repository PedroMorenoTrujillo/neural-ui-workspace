Sidebar

Panel lateral controlado 100% por la URL. El estado abierto/cerrado vive en ?menu=open — compartible, navegable con el historial del navegador, y sincronizado automáticamente via withComponentInputBinding().

URL State input() ← QueryParam computed() 400px Mobile-First

Estado en la URL

El input() menu recibe el valor del queryParam ?menu= gracias a withComponentInputBinding(). Un computed() deriva si el sidebar debe estar abierto.

URL actual
{{ isSidebarOpen() ? 'Sidebar ABIERTO' : 'Sidebar CERRADO' }}
{{ currentUrl() }} 
// app.config.ts — habilitar URL input binding
provideRouter(routes, withComponentInputBinding())

// sidebar-demo.component.ts — input() ← queryParam automático
menu = input<string>();   // ← hydratado desde ?menu=open

readonly isSidebarOpen = computed(() => this.menu() === 'open');

// Abrir: añade ?menu=open a la URL
openSidebar(): void {
  this.router.navigate([], {
    queryParams: { menu: 'open' },
    queryParamsHandling: 'merge',
  });
}

API

Input / Output Tipo Default Descripción
isOpen boolean false Controla visibilidad. Vincular con URL state.
ariaLabel string 'Menú de navegación' Etiqueta accesible del <aside>
closeLabel string 'Cerrar menú...' Etiqueta del botón cerrar para lectores de pantalla
(closeRequested) void — Emite al pulsar overlay o botón cerrar
